1) Protocols, Algorithms and Source Code in C: Chapter 1

   - In terminlogy a sender transmits a message securely to a receiver to prevent eavesdroppers from reading it.
   - Plaintext (or cleartext) is the original message. Encryption changes plaintext to hide its substance.
   - DEscription is the process of reverting ciphertext back into its original plaintext form.
   - Last but not least Cryptology as mentioned in thr book is the branch of mathematics that includes both cryptography (the science of keeping messages secure) and cryptanalysis (the science of breaking ciphertext).
   - Steganography hides the very existence of a secret message within an innocuous one.
   - Algorithms generally use substitution (replacing characters) or transposition (rearranging characters)
   - Some substitution types include, simple substitution where each character is replaced by a corresponding ciphertext character and Polygram which blocks or groups of characters are encrypted together.
   - Simple XOR simplifies to a symmetric algorithm where plaintext is XORed with a keyword to produce ciphertext.
   - Then there is One Time Pads, perfect secrecy, invented in 1917, it uses a large, nonrepeating set of truly random key letters.
   - There are also some computer algorithms like (Data Encryption Standard)
   - Lastly for Cryptographic Context large numbers are used to describe cryptographic scales, like the odds of being killed by lightning (1 in 9 billion).

2) PGP - Send Encrypted and Signed Message - gpg: Karvinen 2023
   - The purpose of PGP and GPG is due to encryption it prevents unauthorized reading of messages, while signing protects messages from being modified.
   - PGP is a highly secure standard for encryption, and gpg (GNU Privacy Guard) is a popular tool used to implement it.
   - There are some tools like the core tool is gpg, but micro (text editor) and psmisc (providing killall) are helpful additions.
   - some general commands include: "gpg --gen-key" which will generates a new keypair and "gpg --fingerprint" that displays the unique fingerprint used to verify a key's authenticity.
   - For the Alice and Tero Simulation, Alice is simulated using a separate folder with its own configuration via the --homedir . flag. Alice must verify that the key’s fingerprint matches Tero's.
   - Alice writes a plaintext message (e.g., message.txt) and then runs a comand, and uses Tero’s public key so only he can read it.
   - After that Tero uses "gpg --decrypt encrypted.pgp" to unlock the message.
   - For trouble shooting If gpg-agent becomes confused during key generation, use "killall gpg-agent" to reset the process.
